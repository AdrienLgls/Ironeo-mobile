## Codebase Patterns

- Screens vivent dans `app/` (pas de sous-dossier `screens/`)
- TabNavigator importe depuis `./ScreenName` (relatif à app/)
- Types dans `types/` — un fichier par domaine (auth.ts, user.ts...)
- Services dans `services/` — pattern : `const { data } = await api.get<Type>('/endpoint'); return data`
- NativeWind : `bg-background` (#121212), `text-accent` (#EFBF04), `bg-white/[0.05]` pour surfaces
- ScrollView avec `contentContainerClassName` pour le padding interne
- État local : useState + useEffect dans le screen
- Pattern erreur : `.catch(() => setError('message')).finally(() => setLoading(false))`
